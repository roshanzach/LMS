# Contributing to UniLMS

Thank you for your interest in contributing to UniLMS! We appreciate your help in building a production-grade LMS for engineering institutions.

## Code of Conduct

Please be respectful and professional in all communication and code reviews.

## Git & Workflow Guidelines

1. Fork the repository and create your branch from `main`.
2. Follow the monorepo structure:
   - Frontend changes -> `apps/web`
   - Backend logic -> `apps/api`
   - Shared interfaces -> `packages/types`
   - DB migrations -> `packages/database`
3. Write clean, documented TypeScript.
4. Ensure files are formatted and linted before submitting.
5. Create a descriptive Pull Request referencing the issue you are fixing.
